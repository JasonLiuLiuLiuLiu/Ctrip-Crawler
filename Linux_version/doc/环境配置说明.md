# Ctrip Flight Scraper - 环境配置说明

## 项目概述

这是一个携程航班数据爬虫项目，用于自动化采集航班信息。本文档详细说明了在 Ubuntu 24.04 Desktop (minimal install) 上运行此项目所需的所有依赖和配置步骤。

## 系统要求

### 操作系统
- **推荐**: Ubuntu 24.04 Desktop (minimal install 或完整安装均可)
- **支持**: Ubuntu 20.04+, Debian 11+, 其他基于 Debian 的 Linux 发行版

### 硬件要求
- **CPU**: 双核或以上
- **内存**: 至少 2GB RAM（推荐 4GB+）
- **存储**: 至少 5GB 可用空间
- **网络**: 稳定的互联网连接（支持 IPv6 更佳）

## 快速安装指南

### 方式一：使用自动化脚本（推荐）

1. **下载项目并进入目录**
```bash
cd /home/ubuntu/Ctrip-Crawler/Linux_version
```

2. **赋予脚本执行权限**
```bash

```

3. **运行安装脚本（需要 sudo 权限）**
```bash
sudo bash setup_environment.sh
```

脚本会自动完成：
- ✓ 系统更新
- ✓ 安装所有系统依赖
- ✓ 安装 Python 3 和 pip
- ✓ 安装 Chrome 浏览器
- ✓ 安装 ChromeDriver
- ✓ 配置网络工具
- ✓ 创建 Python 虚拟环境
- ✓ 安装 Python 依赖包
- ✓ 创建必要的目录
- ✓ 生成配置文件
- ✓ 创建运行脚本

4. **等待安装完成**

安装过程可能需要 5-15 分钟，取决于你的网络速度。

### 方式二：手动安装

如果你想了解每个步骤或需要自定义安装，可以参考以下手动安装步骤：

#### 步骤 1: 更新系统
```bash
sudo apt-get update
sudo apt-get upgrade -y
```

```bash
sudo apt-get install -y build-essential software-properties-common \
    apt-transport-https ca-certificates curl wget gnupg git unzip
```

#### 步骤 3: 安装 Python 3
```bash
sudo apt-get install -y python3 python3-pip python3-dev python3-venv
python3 -m pip install --upgrade pip
```

#### 步骤 4: 安装系统库
```bash
sudo apt-get install -y libmagic1 libmagic-dev file \
    libxml2-dev libxslt1-dev zlib1g-dev
```

#### 步骤 5: 安装 Chrome
```bash
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt-get install -y ./google-chrome-stable_current_amd64.deb
rm google-chrome-stable_current_amd64.deb
```

#### 步骤 6: 安装 ChromeDriver
```bash
sudo apt-get install -y chromium-chromedriver
```

#### 步骤 7: 安装网络工具
```bash
sudo apt-get install -y iproute2 iptables net-tools iputils-ping dnsutils
```

#### 步骤 8: 创建虚拟环境并安装 Python 依赖
```bash
cd /home/ubuntu/Ctrip-Crawler/Linux_version
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
deactivate
```

## 配置项目

### 1. 创建配置文件

```bash
cd /home/ubuntu/Ctrip-Crawler/Linux_version
cp config.json.example config.json
nano config.json
```

### 2. 主要配置项

#### 爬取配置 (scraping)
```json
{
  "scraping": {
    "crawl_cities": ["上海", "香港", "东京"],  // 要爬取的城市
    "crawl_days": 60,                          // 爬取天数
    "begin_date": null,                        // 开始日期（可选）
    "end_date": null,                          // 结束日期（可选）
    "start_interval": 1,                       // T+N 天
    "days_interval": 1,                        // 日期间隔
    "crawl_interval": 1,                       // 请求间隔（秒）
    "max_wait_time": 10,                       // 最大等待时间（秒）
    "max_retry_time": 5,                       // 最大重试次数
    "direct_flight_only": true,                // 只爬取直飞航班
    "include_comfort_data": false              // 包含舒适度数据
  }
}
```

#### 代理配置 (proxy)
```json
{
  "proxy": {
    "enabled": true,                           // 是否启用代理
    "ipv6_count": 120,                         // IPv6 接口数量
    "base_interface": "eth0",                  // 基础网络接口
    "ip_mode": "normal",                       // 代理模式: normal/random
    "delete_interface": false,                 // 是否删除接口
    "proxy_port": 1080,                        // 代理端口
    "proxy_bind_address": "0.0.0.0"            // 代理绑定地址
  }
}
```

**注意**: 如果你的服务器不支持 IPv6 或不需要使用代理，请设置 `"enabled": false`

#### 登录配置 (login)
```json
{
  "login": {
    "allowed": true,                           // 是否允许登录
    "accounts": ["your_account"],              // 账号列表
    "passwords": ["your_password"],            // 密码列表
    "cookies_file": "cookies.json"             // Cookie 文件路径
  }
}
```

#### 调试配置 (debug)
```json
{
  "debug": {
    "enable_screenshot": false,                // 是否启用截图
    "log_level": "INFO",                       // 日志级别: DEBUG/INFO/WARNING/ERROR
    "log_file": null,                          // 日志文件路径（null 表示只输出到控制台）
    "headless": true                           // 是否使用无头模式（false 则显示浏览器窗口）
  }
}
```

## 常见问题排查

### 1. Chrome/ChromeDriver 版本不匹配

**症状**: 运行时报错 `session not created: This version of ChromeDriver only supports Chrome version XX`

**解决方案**:
```bash
# 查看 Chrome 版本
google-chrome --version

# 重新安装 ChromeDriver
sudo apt-get install --reinstall chromium-chromedriver
```

### 2. 缺少系统库

**症状**: 安装 Python 包时报错 `error: command 'gcc' failed`

**解决方案**:
```bash
sudo apt-get install build-essential python3-dev
```

### 3. libmagic 找不到

**症状**: 运行时报错 `ImportError: failed to find libmagic`

**解决方案**:
```bash
sudo apt-get install libmagic1 libmagic-dev
```

### 4. 权限错误

**症状**: 创建网络接口时报错 `Permission denied`

**解决方案**: 代理功能需要 root 权限
```bash
sudo python main.py
# 或者
sudo ./run.sh
```

### 5. IPv6 不可用

**症状**: 代理启动失败，提示 IPv6 相关错误

**解决方案**: 禁用代理功能
```json
{
  "proxy": {
    "enabled": false
  }
}
```

或者启用 IPv6:
```bash
sudo sysctl -w net.ipv6.conf.all.disable_ipv6=0
sudo sysctl -w net.ipv6.conf.default.disable_ipv6=0
```

### 6. 虚拟环境激活失败

**症状**: `source venv/bin/activate` 不工作

**解决方案**:
```bash
# 重新创建虚拟环境
cd /home/ubuntu/Ctrip-Crawler/Linux_version
rm -rf venv
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## 运行模式说明

### 无头模式 vs 有头模式

程序默认使用 **无头模式（Headless）** 运行，可以通过配置切换。

#### 无头模式（推荐）
**配置**: `"headless": true`

**特点**：
- ✅ 浏览器在后台运行，不显示图形界面
- ✅ 占用资源少，适合服务器环境
- ✅ 适合批量爬取、长时间运行
- ❌ 无法看到浏览器实际操作过程

**适用场景**：
- 生产环境运行
- 服务器批量采集
- 无图形界面的环境

#### 有头模式（调试用）
**配置**: `"headless": false`

**特点**：
- ✅ 显示浏览器窗口，可以看到操作过程
- ✅ 便于调试和问题排查
- ❌ 占用更多资源
- ❌ 需要图形界面环境

**适用场景**：
- 开发调试
- 问题排查
- 验证爬取逻辑

**配置示例**：
```json
{
  "debug": {
    "enable_screenshot": true,
    "log_level": "DEBUG",
    "log_file": "logs/debug.log",
    "headless": false  // 启用有头模式，显示浏览器窗口
  }
}
```

**注意事项**：
- Ubuntu Server 或 minimal install 需要安装图形界面才能使用有头模式
- 远程服务器可以使用 X11 forwarding 或 VNC 查看
- 或者使用 Xvfb 虚拟显示：`xvfb-run -a ./run.sh`

## 性能优化建议

### 1. 提高爬取速度
- 增加 `crawl_interval` 间隔可以提高成功率
- 减小 `max_wait_time` 可以减少等待时间
- 禁用 `include_comfort_data` 可以减少数据处理时间

### 2. 降低资源占用
- 使用 headless 模式（默认已启用）
- 禁用图片加载（默认已禁用）
- 减少 `ipv6_count` 数量

### 3. 提高稳定性
- 增加 `max_retry_time` 重试次数
- 启用日志记录便于调试
- 使用多个账号轮换

## 安全注意事项

1. **不要将密码硬编码到代码中**
   - 使用 `config.json` 存储配置
   - `config.json` 应添加到 `.gitignore`

2. **遵守法律法规**
   - 遵守网站的 robots.txt 和服务条款
   - 控制爬取频率，避免对服务器造成压力
   - 仅用于学习和研究目的

3. **账号安全**
   - 使用专门的测试账号
   - 定期更换密码
   - 不要分享包含凭据的配置文件

4. **网络安全**
   - 在受信任的网络环境中运行
   - 使用防火墙保护代理端口
   - 定期更新系统和依赖包
